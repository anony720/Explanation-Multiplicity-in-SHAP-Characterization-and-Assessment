# Explanation Multiplicity in SHAP: Characterization and Assessment

---

## Overview

We study **explanation multiplicity**: the phenomenon where a single prediction admits multiple valid but mutually inconsistent SHAP explanations, even when the dataset, model class, and trained model are all fixed. We characterize two sources вҖ” background-sampling randomness and model-seed randomness вҖ” and introduce calibrated disagreement metrics (в„“вӮӮ, Top-k Jaccard, RBO) to quantify their magnitude.

## Reproducing Experiments

### Step 1 вҖ” Prepare datasets

German, Diabetes, and ACS data are included in `dataset/`. To generate GMSC:

```bash
python prepare_gmsc.py
```

### Step 2 вҖ” Download precomputed SHAP results

The main SHAP results (~800 MB) are hosted on Google Drive. Download and extract them into the repository root:

```bash
# Download
gdown "https://drive.google.com/uc?id=1Hr5SikYiuKHwobmFQlatPKwaY02VswOe" -O results_tree.tar.gz

# Extract
tar -xzf results_tree.tar.gz

# (Optional) remove the archive
rm results_tree.tar.gz
```

This creates a `results_tree/` directory containing all precomputed SHAP value files.
`results_true/` (full-background KernelSHAP) and `proba/` (prediction probabilities) are already included in this repository.

If you prefer to run experiments from scratch, skip this step and proceed to Step 3.

### Step 3 вҖ” Run experiments

**Main experiment** (background sampling sensitivity):

```bash
python experiment_sampling.py \
    --dataset german \
    --model ftt \
    --split_seed 0 \
    --model_seed_idx 0 \
    --explainer kernel_shap
```

Arguments:
| Argument | Options | Description |
|---|---|---|
| `--dataset` | `german`, `diabetes`, `acs`, `gmsc` | Dataset |
| `--model` | `dt`, `rf`, `xgb`, `mlp`, `ftt` | Model class |
| `--split_seed` | 0вҖ“4 | Cross-validation fold |
| `--model_seed_idx` | 0вҖ“4 | Index into model seed list `[0,21,42,63,84]` |
| `--explainer` | `kernel_shap`, `permutation_shap` | Explainer type |
| `--bg_size` | int (default 100) | Background set size |
| `--n_chunks` / `--chunk_idx` | int | Chunk test set for parallelism (large datasets) |
| `--only_method` | `random`, `kmeans`, `pred_stratified`, `cte` | Run a single background method |

To run all dataset Г— model combinations locally:

```bash
bash run_sampling.sh
```

**TreeSHAP reference** (DT and RF only):

```bash
python experiment_tree.py --dataset german --model dt --split_seed 0
```

**Full-background KernelSHAP** (ground truth):

```bash
python experiment_true.py --dataset german --model ftt --split_seed 0
```

**Prediction probabilities** (required for certainty-stratified analysis):

```bash
python experiment_proba.py --dataset german --model ftt --split_seed 0
```

Results are saved to `results_tree/` (bg_size=100) or `background_results/` (other bg sizes).

### Step 4 вҖ” Generate figures

Open and run the analysis notebooks in order:

```bash
jupyter notebook background_sampling.ipynb   # Figure 1: sensitivity by method / model
jupyter notebook background_true.ipynb       # Figure 2: distance to true SHAP
jupyter notebook background_table.ipynb      # Table: summary statistics
```

Each notebook has a `DATASET` variable in the first cell; set it to `german`, `diabetes`, `acs`, or `gmsc` and run all cells. Figures are saved to `figures_bg/`.

## Metrics

| Metric | Description |
|---|---|
| в„“вӮӮ distance | Mean per-instance Euclidean distance between SHAP vectors |
| Top-k Jaccard distance | 1 вҲ’ Jaccard similarity of top-k feature sets (k=3) |
| 1 вҲ’ RBO | Rank-biased overlap disagreement between ranked feature lists |

Calibration baselines (randomized null models) are computed via `baseline.py` / `plot_violin.py`.
